# Project Specification: Mario Super Sluggers Custom Netplay Architecture

## 1. Objective
Achieve LAN-like online multiplayer for Mario Super Sluggers using physical Wii Remotes. Bypass standard Dolphin Bluetooth passthrough limitations to eliminate fixed-frame delay and network jitter desynchronization.

## 2. Infrastructure & Hardware
* **Server/Host:** 2016 MacBook Pro running Linux Mint (Gigabit Ethernet).
* **Clients:** Standard PCs.
* **Controllers:** Physical Wii Remotes connected via standard local Bluetooth adapters.

## 3. Core Architecture
The system replaces standard emulator netplay with a Custom Hybrid Input Proxy System utilizing dynamic host authority.

### Phase 1: Engine Reverse Engineering
* **Target:** `main.dol` and `.rel` files extracted from the Wii ISO.
* **Action:** Decompile to `.elf` using Ghidra/IDA Pro.
* **Goal:** Identify raw PowerPC memory addresses for specific game states: Pitching wind-up, batter swing threshold, and ball-in-play flags.
* **Implementation:** Write custom Gecko codes (assembly hooks) to trigger network state changes based on these memory addresses.

### Phase 2: High-Frequency Bluetooth Proxy (Local Background Service)
* **Language:** C++ or Rust (requires low-level HID/kernel access).
* **Function:** Poll local Bluetooth adapter at 1000Hz (1ms intervals).
* **Data Handling:** Capture raw accelerometer, gyroscope, IR pointer coordinates, and digital button states.
* **Processing:** Interpolate dropped polling frames to prevent desynchronization. Serialize data into UDP packets stamped with high-resolution microsecond timing.

### Phase 3: Custom Dolphin Emulator Fork
* **Input Injection:** Bypass standard "Real Wii Remote" passthrough. Build a custom API to ingest the proxy's UDP packets, forcing them into Dolphin's "Emulated Wii Remote" framework.
* **Adaptive Jitter Buffering:** Replace fixed-frame buffers with a time-warped adaptive buffer. Use linear interpolation on late packets to smooth IR pointer paths across network jitter without pausing the rendering loop.
* **Dynamic Host Authority (Netcode):**
    * *Pitching Phase:* Batter holds 0ms host authority. Pitcher inputs are transmitted early.
    * *Fielding Phase:* Engine hooks (from Phase 1) instantly transfer host authority to the fielding team the millisecond the ball is hit.

### Phase 4: Automated Launcher
* **Framework:** Electron or Tauri.
* **Isolation:** Generate local `portable.txt` and temporary `.ini` files to isolate settings from the user's primary Dolphin installation.
* **Lifecycle:**
    1. Initialize background Bluetooth proxy.
    2. Launch Dolphin fork directly into the ISO via CLI arguments.
    3. Monitor process and automatically terminate proxy upon game exit.
